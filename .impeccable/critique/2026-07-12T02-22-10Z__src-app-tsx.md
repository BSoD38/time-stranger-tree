## Anti-Patterns Verdict

**Does this look AI-generated? Yes — at the identity level, not the code level.**

**LLM assessment:** Textbook "AI dark data-tool" reflex. First-order category check fails outright — you can guess the whole palette from "graph explorer tool" without opening the repo: near-black-blue bg (`#0a0e14`), one cyan/teal accent (`#00e5c3`), a geometric HUD display font (Chakra Petch), hairline pill chips. Second-order fails too: it's the predictable "data tool that isn't light-mode → dark terminal + cyan + techy font" family, and it doesn't even diverge within the cliché. The specific tells: the **blueprint grid background** (`tokens.css:37-40`, the most on-the-nose "engineering tool" signifier there is), the HUD typeface, cyan-on-black, uppercase tracked micro-labels, tabular-nums everywhere. **Emotional-fit failure is the real problem:** *Digimon* is vibrant, character-driven, nostalgic — and the one genuine color source, the monster **sprites**, is muted to 8–12% opacity behind a cold grid. The design treats beloved characters as data points on an oscilloscope.

**Deterministic scan:** `detect.mjs` returned **0 findings** across five invocations (incl. a raw `--no-config` pass), verified functional by a positive-control fixture (which correctly flagged 3 tells). This is the key tension: **the code-level hygiene is genuinely clean** — no gradient text, no side-stripes (fixed in the recent extract pass), no bounce easing, no overused fonts, disciplined token/z-index/radius scales. The scanner and the design director *disagree*, and both are right: the slop is 100% at the **aesthetic-identity** layer, which no deterministic rule can see.

**Visual overlays:** Unavailable — no browser automation tool is exposed, so no user-visible overlay was produced. Findings above are from source review + the CLI detector.

## Overall Impression

Solid information-design bones and real power-user depth, wearing the single most predictable "AI dark data-tool" skin, over subject matter that is crying out for warmth and character. The good news, and it's substantial: **the slop is skin-deep.** The token system is clean enough that a re-skin is mostly retokenizing; the graph layering, the focus/route mechanics, the keyboard accelerators, and the URL-sync are all worth keeping. Biggest single opportunity: **let the sprites lead and give the app a Digimon-native identity** instead of a HUD.

## What's Working

1. **Disciplined token system.** Documented radius scale (`--radius-xs` "sprite thumbnails" → `--radius-lg` "header portrait"), a purposeful commented z-index scale, one source of truth for attribute color shared between JS and the Cytoscape stylesheet. This makes a redesign *cheap*.
2. **The graph styling is real information design, not decoration.** Layered dim/highlight precedence (`dim-filter < dim-soft < dim-hard < route-glow < sel`), `underlay` glow for selection instead of a crude border, edge *semantics* by line style (jogress dashed purple, item dash-pattern amber, bond dashed cyan), generation traits encoded structurally (armor = double border, hybrid = dashed).
3. **Power-user affordances are coherent.** `/` & Ctrl+K search, arrow/Enter/Shift+Enter in the dropdown, F to focus, Esc to unwind, double-tap-to-focus, shareable URL state, a "use selected" bridge between panel and route planner.

## Priority Issues

**[P0] No identity / wrong emotional register for the subject.**
- **Why it matters:** The audience — broad, nostalgic *Digimon* fans — feels nothing on arrival; it reads as an internal engineering tool. This is the "looks like AI slop" complaint, literally.
- **Fix:** Commit to a Digimon-native identity. Let the sprites be the color source and quiet the chrome; drop the blueprint grid; replace Chakra Petch's HUD affect with a display face (or wordmark) that reads playful/adventurous; choose a committed palette (in-world device / field-guide / sprite-adaptive). Make dark-vs-light a *choice*, not a reflex.
- **Suggested command:** /impeccable colorize + /impeccable typeset + /impeccable delight (a full identity redesign).

**[P1] Cold, guidance-free first run.**
- **Why it matters:** On load with no selection the right panel is `null` (`App.tsx:119`); new users get a wall of 475 tiny tiles and must *guess* that clicking or searching does anything. The best features (focus, route) are invisible.
- **Fix:** A welcoming empty panel — "Pick a Digimon to see its lineage," a few featured Digimon chips, a compact key of the glyphs + shortcuts. A first-run hint over the canvas.
- **Suggested command:** /impeccable onboard.

**[P2] Meaning conveyed by color alone.**
- **Why it matters:** Graph nodes encode attribute *only* by 7 border hues (incl. the blue/green/purple confusion set); resistances lead with red/green. ~8% of male users can't reliably separate these. Fails WCAG 1.4.1.
- **Fix:** Add a redundant channel — a small attribute letter/glyph badge on nodes (you already do this for generation via border-style); reinforce resistance cells with an icon so color isn't load-bearing.
- **Suggested command:** /impeccable audit → /impeccable colorize.

**[P2] Keyboard-inoperable surfaces + no focus-visible.**
- **Why it matters:** The Cytoscape graph has no keyboard path; the route endpoint dropdown & "use selected" fire on `onMouseDown` only; there is **zero** `:focus-visible` styling anywhere (only `outline:none` + a 1px border swap). Keyboard/SR users select from search then hit dead ends. Fails WCAG 2.1.1 / 2.4.7.
- **Fix:** A tokenized `:focus-visible` ring on every interactive element; real click/keydown handlers on the route dropdown; a minimal keyboard path into the graph (roving focus / jump-and-pan).
- **Suggested command:** /impeccable audit.

**[P3] Raw error text on load failure.**
- **Why it matters:** `App.tsx:64-68` renders `String(err)` on the splash with no recovery — a dead end for a non-technical fan.
- **Fix:** Friendly message + Retry button; log the raw error to console only.
- **Suggested command:** /impeccable harden.

## Persona Red Flags

**Alex (power user)** — well served (`/`, Ctrl+K, F, Esc, double-tap, URL sync). Gaps: **no shortcut discoverability** (only a tiny line in the search dropdown; no `?` cheatsheet), and the **graph + route dropdown aren't keyboard-drivable**, inconsistent with the fully-keyboard main search.

**Sam (accessibility)** — Contrast is a pleasant surprise: `--text-dim #8b98a9` ≈ 5.5–6.6:1 on the surfaces (passes AA) — **but** most of it renders at 9.5–11px, so it clears contrast while failing on legibility. **No `:focus-visible` anywhere** — custom buttons rely on unreliable UA rings on `#0a0e14` (fails 2.4.7). Graph + route dropdown not keyboard-operable (fails 2.1.1). Attribute on nodes and the search-row attribute dot are color-only. (Evolve/devolve is done *right* — redundant via dashed line + reversed arrow + ▲/▼ glyph + label.)

**Riley (edge cases)** — **0 filter results** shows "0 / 475" with every node faded to 12% — looks like a render bug, not "no matches." **0 search results** renders nothing (no "No Digimon found"). Long names in the `inline` MonRow variant (`overflow: visible`) can blow out route A→B rows; graph node labels don't wrap and can overlap at zoom. The **14,700px-tall mouse-only canvas** has no minimap / "scroll to top" / progress sense. Route-not-found, in contrast, is the best-handled edge case in the app.

## Minor Observations

- **Glyph soup with no key:** ◈ ⧉ ❖ ◆ 🐎 ▲/▼ — meanings live only in `title` tooltips.
- **Two search UIs, different capabilities** — unify the route EndpointPicker with the main SearchBox.
- **Description "click to expand"** (`DetailPanel.tsx:73-79`) has no visible "more" affordance.
- **Legend is `pointer-events:none`** — can't double as a filter (missed easy win).
- **`--danger` is overloaded** as both "weak resistance" and "destructive/clear" — a weakness isn't an error.
- **Styled scrollbar is WebKit-only**; Firefox gets defaults.
- **`prefers-reduced-motion` is handled** — good, though there's barely any motion to reduce, which is itself telling.
- **Brand is a stock 🌳 emoji favicon** — the tab presence is as generic as the UI.

## Questions to Consider

1. What if the **sprites were the design system, not decoration** — the palette/accent/panel tint derived from the *selected* Digimon's dominant colors, so the whole UI shifts as you explore?
2. Why is this a "graph explorer" *visually* at all? What if the frame were an **in-world device** (a Digivice / Digital Gate) rather than an analytics dashboard?
3. Can the design **earn a celebration** at its emotional peak (reaching a Mega, completing a route)? Nostalgia games run on the dopamine of evolution; the UI has none.
4. Could the tree read as a **living lineage** (organic growth, legible generation bands) rather than a schematic node graph — keeping the 🌳 promise the favicon makes?
5. Delete the grid, swap Chakra Petch, let one Digimon's art set the accent — would anyone still say "AI made this"? (My read: no. The identity problem is skin-deep, which is the good news.)
